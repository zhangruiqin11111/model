1、本地安装GIT 了解GIT工作原理、机制，可实现代码的同步、上传等操作

2、配置GIT (https://www.cnblogs.com/cxscode/p/8325064.html)
	我们先在电脑硬盘里找一块地方存放本地仓库，比如我们把本地仓库建立在C:\MyRepository\1ke_test文件夹下,
	进入1ke_test文件夹 鼠标右键操作如下步骤：
	a.在本地仓库里右键选择Git Init Here，会多出来一个.git文件夹，这就表示本地git创建成功。
	右键Git Bash进入git命令行,为了保险起见，我们先执行git init命令
	$ git init
	b.在本地创建ssh key
	$ ssh-keygen -t rsa -C "your_email@youremail.com"
	后面的your_email@youremail.com改为你的邮箱。也是在github上注册的那个邮箱
	将生成的key传到GITHUB 项目上,验证是否成功，在git bash下输入 $ ssh -T git@github.com
	注：添加ssh key原因：
		https 和 SSH 的区别：
		1、前者可以随意克隆github上的项目，而不管是谁的；而后者则是你必须是你要克隆的项目的拥有者或管理员，且需要先添加 SSH key ，否则无法克隆。
		2、https url 在push的时候是需要验证用户名和密码的；而 SSH 在push的时候，是不需要输入用户名的，如果配置SSH key的时候设置了密码，则需要输入密码的，
	否则直接是不需要输入密码的。
	c.接下来我们要做的就是把本地仓库传到github上去，在此之前还需要设置username和email，
	因为github每次commit都会记录他们
	$ git config --global user.name "your name"
	$ git config --global user.email "your_email@youremail.com"
	d.进入要上传的仓库，右键git bash，添加远程地址
	$ git remote add origin git@github.com:yourName/yourRepo.git(后面的yourName和yourRepo表示你再github的用户名和刚才新建的仓库，加完之后进入.git，打开config，
	这里会多出一个remote “origin”内容，这就是刚才添加的远程地址，也可以直接修改config来配置远程地址。)

3、提交上传
	1）接下来在本地仓库里添加一些文件，比如README，在本地新建一个README文件
	然后在命令行输入一下命令
	$ git add README
	$ git commit -m "first commit"
	2）上传到github 
	$ git push origin master
	$ git push命令会将本地仓库推送到远程服务器。
	$ git pull命令则相反。

	注：首次提交，先git pull下，修改完代码后，使用git status可以查看文件的差别，使用git add 添加要commit的文件。
	
4、GIT查看远程仓库连接
	使用git remote -v  查看远程仓库地址

5、GIT删除本地仓库的文件或文件夹
	$ dir // 查询跟目录的所有文件和文件夹
	$ dir test // 查询test文件夹下的所有文件和文件夹
	$ git rm -r --cached test/a.txt // 删除test文件夹下的a.txt文件，加入到删除缓存中
	$ git commit -m '删除test文件夹下的a.txt文件' // 执行删除缓存，提交文字为“删除test文件夹下的a.txt文件”
	$ git rm -r --cached test // 删除test文件夹，加入到删除缓存中
	$ git commit -m '删除test文件夹' // 执行删除缓存，提交文字为“删除test文件夹”
	$ git push // 推送操作到远程仓库

6、GIT删除远程仓库的文件或文件夹
	$ git pull origin master  本地同步远程仓库，将远程仓库里的内容拉下来
	$ git rm -r --cached 文件名   删除文件
	$ git commit -m '删除test文件夹'   提交并添加说明
	$ git push  将本次更改更新到github项目上去

7、GIT文件的修改与提交
	a、查看文件版本库中的文件状态：
	$ git status
	b.修改文件，再次查看版本库中文件的状态： 
	c.对比文件修改前和修改后的变化：$git diff
	d.提交修改内容与提交新文件是一样的两步
 		$ git add 文件名
		$ git commit -m "add a line"
	e.提交后，再次查看版本库当前状态，会看到文件没有需要提交的更改。
	f.撤销操作 
	有时候，我们提交完成后发现漏掉几个文件没有添加，或者信息写错了，这时候就需要对操作进行撤销。
	$ git commit --amend
	$ it commit -m 'xxxxx'
	$ git add file
	$ git commit --amend	
	g.撤销对文件的修改
	如果不想保留对某一个文件的修改，则可以使用git checkout -- filename来撤销对文件的修改。
	但是这个命令只是拷贝了另一个文件来覆盖它，文件的所有更改都将消失。

8、git怎么回退到某个历史commit
	$ git log #查看
	在Terminal 或者git控制条 执行 回退到某个版本命令
	$ git reset --hard 139dcfaa558e3276b30b6b2e5cbbb9c00bbdca96 

9、git取消merger状态：
	$ git reset --hard
